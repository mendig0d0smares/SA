Fundamentos de Redes

O que são Redes de Computadores?

Redes de Computadores são um conjunto de dispositivos interligados (como computadores, celulares, servidores, impressoras, roteadores etc.) que compartilham informações, serviços e recursos entre si por meio de algum tipo de comunicação — geralmente cabos ou conexões sem fio.

O que é uma Rede mínima?

Rede mínima é o menor conjunto de dispositivos e conexões necessários para formar uma rede de computadores funcional. Ou seja, é a estrutura mais simples possível que permite a comunicação entre dois ou mais equipamentos..

O que é a internet?

A Internet é uma rede global que interliga milhões de computadores e dispositivos no mundo todo, permitindo a troca de informações, o acesso a sites, vídeos, aplicativos, serviços online e muito mais. Ela funciona como uma enorme rede de redes, conectando desde pequenos dispositivos domésticos até grandes servidores e empresas.

O que compõe?

A Internet é formada por diversos elementos que trabalham juntos. Aqui estão os principais:

Dispositivos (hosts)

São os equipamentos que enviam e recebem dados.

Provedores de Internet (ISPs)

São as empresas que fornecem acesso à internet.

Roteadores e Switches

Equipamentos responsáveis por direcionar o caminho dos dados pela rede. Eles garantem que a informação chegue ao destino correto.

Servidores

Computadores muito potentes que guardam e entregam: Sites, Vídeos, Arquivos, Jogos, aplicativos

Protocolos de comunicação

São regras que permitem os dispositivos se entenderem:

TCP/IP – base da internet

HTTP/HTTPS – para acessar sites

DNS – converte nomes (google.com) em endereços IP

SMTP/POP/IMAP – e-mails

Sem protocolos, os dispositivos não conseguiriam se comunicar.

Meios de transmissão

ão as formas pelas quais os dados viaja.

O que é web?

A Web é um conjunto de documentos interligados (páginas, textos, imagens, vídeos, sistemas).

O que compõe a Web?

1. Navegadores (Browsers)

Programas usados para acessar páginas da Web:

2. Servidores Web

Computadores que armazenam sites e entregam o conteúdo quando alguém acessa.

3. Linguagens da Web

CSS → aparência e design

JavaScript → interatividade

PHP, Python, Node.js → funcionamento no servidor

O que é o mapa de fibra óptica.

O mapa de fibra óptica é uma representação visual (um mapa) que mostra por onde passam os cabos de fibra óptica que conectam países, continentes e cidades ao redor do mundo. Ele permite ver como a Internet global está interligada através de cabos submarinos e terrestres feitos de fibra óptica — a tecnologia que transmite dados em altíssima velocidade usando luz.

Componentes de Hardware de uma Rede Wi-Fi

Roteador Wi-Fi

É o equipamento central da rede sem fio.

Funções:

Cria e transmite o sinal Wi-Fi.

Distribui endereços IP.

Roteia o tráfego entre os dispositivos e a internet.

Modem

Dispositivo que recebe a conexão do provedor (fibra, cabo, DSL, rádio) e entrega internet ao roteador.

Ponto de Acesso (Access Point – AP)

Usado para expandir o alcance da rede Wi-Fi, especialmente em empresas, escolas e casas grandes. Ele cria uma nova área de cobertura wireless usando a rede cabeada.

Repetidor / Extensor Wi-Fi

Amplifica o sinal do roteador para lugares mais distantes. Não cria nova rede, apenas repete o sinal existente.

Switch (opcional)

Equipamento usado quando há muitos dispositivos cabeados. Permite ligar vários APs ou computadores via cabo em uma rede Wi-Fi corporativa.

Antenas Wi-Fi

Podem ser internas ou externas. Servem para:

Melhorar o alcance do sinal

Direcionar ou ampliar cobertura

Placas de Rede Sem Fio (Wi-Fi Adapter)

São os dispositivos que permitem que computadores e outros aparelhos se conectem à rede Wi-Fi.

Controladora de Rede (em redes grandes)

Usada em empresas para gerenciar vários pontos de acesso ao mesmo tempo.

Componentes de Hardware de uma Rede Local com Cabo (LAN Cabeada)

Uma rede local cabeada utiliza cabos (geralmente par trançado, como Cat5e, Cat6, Cat6a, Cat7) para conectar computadores e dispositivos. Os principais componentes de hardware são:

Switch (Comutador) – o centro da rede

É o equipamento principal de uma LAN cabeada. Funções:

Conectar vários dispositivos usando portas RJ-45

Encaminhar dados entre computadores

Organizar o tráfego da rede

Aumentar o desempenho e a estabilidade

Sem um switch, uma rede cabeada não existe.

Roteador (Opcional na LAN interna)

Usado quando a rede precisa ter acesso à Internet. Ele conecta a LAN ao modem do provedor.

Modem

Quando a LAN precisa acessar a internet, o modem:

Recebe a conexão do provedor (fibra, cabo, DSL)

Entrega o sinal ao roteador

Cabos de Rede (Par Trançado Ethernet)

Usados para conectar PCs, switches e roteadores.

Computadores e Dispositivos com Placa de Rede (NIC)

Para se conectar a uma rede cabeada, os equipamentos precisam de uma:

Placa de rede Ethernet integrada

Ou adaptador USB-Ethernet

Rack de Rede (em instalações profissionais)

Armazena e organiza:

Patch panels

Switches

O que é um protocolo de comunicação?

Um protocolo de comunicação é um conjunto de regras que permite que dois ou mais dispositivos troquem informações de maneira correta e organizada. Ele funciona como um idioma comum que todos os equipamentos precisam seguir para conseguir se entender. Sem essas regras, cada dispositivo enviaria dados de um jeito diferente, e nenhum saberia interpretar o que o outro quis dizer.

Esses protocolos definem como a comunicação começa, como os dados são enviados, como devem ser lidos e até como corrigir possíveis erros durante o processo. Eles também determinam o formato das mensagens, a sequência em que devem ser transmitidas e como o dispositivo deve agir caso algo saia errado. É graças a esses padrões que computadores, celulares, servidores e roteadores conseguem conversar entre si, mesmo sendo de marcas e tecnologias diferentes.

Protocolos como TCP/IP, HTTP ou HTTPS são usados todos os dias quando acessamos sites, enviamos mensagens, assistimos vídeos ou realizamos qualquer atividade na internet. Portanto, sem protocolos de comunicação, não existiria internet nem qualquer forma de rede funcionando corretamente.

O que é o modelo OSI?

UO modelo OSI é uma forma de explicar como os dados viajam entre dispositivos em uma rede, dividindo esse processo em sete camadas, cada uma com uma função específica. Ele serve como um guia para entender e padronizar a comunicação entre computadores, ajudando diferentes sistemas e equipamentos a funcionarem juntos.

oque é o modelo TCP/IP?

O modelo TCP/IP é o conjunto de regras e camadas que define como a comunicação na internet realmente funciona. Diferente do modelo OSI, que é teórico, o TCP/IP é prático e usado em todos os dispositivos conectados à rede, como celulares, computadores, servidores e roteadores.

Ele organiza a transmissão de dados em quatro camadas, que tratam desde o envio físico dos pacotes até a comunicação entre aplicativos. O modelo recebe esse nome por causa de dois protocolos essenciais: TCP, que garante que os dados cheguem com segurança e na ordem correta, e IP, que cuida dos endereços e da entrega dos pacotes ao destino.

O que é o protocolo IP V4 ?

IPv4 é o protocolo de endereçamento tradicional da internet, mas está ficando limitado por falta de endereços.


O que é o protocolo IP V6 ?

IPv6 é a versão atual do protocolo IP, criada para substituir o IPv4 e oferecer muito mais endereços e melhorias de segurança e desempenho.

Como é configurado o IP e o TCP em uma programação backend com node.js?

Em Node.js você não configura o IP ou o TCP diretamente. Isso é feito pelo sistema operacional. O que você faz na programação é apenas escolher em qual IP e porta o servidor vai escutar.

Quando você cria um servidor HTTP ou TCP em Node, ele automaticamente usa o protocolo TCP/IP por baixo.

Exemplo simples:

server.listen(3000, '127.0.0.1');

3000 é a porta TCP

127.0.0.1 é o IP onde o servidor vai responder

Ou seja: você só define IP e porta; o restante do funcionamento do TCP/IP o Node faz automaticamente.

O que é o protocolo HTTP?

O HTTP (HyperText Transfer Protocol) é o protocolo usado para comunicação entre navegadores e servidores na web.

Ele define como as mensagens são enviadas e recebidas quando você acessa um site, clica em um link, envia um formulário ou carrega uma página.

O HTTP funciona como uma conversa entre cliente e servidor, onde ambos trocam mensagens usando regras padronizadas.

O que é o protocolo HTTPS?

HTTPS é a versão segura do protocolo HTTP.

Enquanto o HTTP envia dados de forma simples, o HTTPS adiciona criptografia, garantindo que ninguém consiga ler, alterar ou interceptar as informações transmitidas entre o navegador e o servidor.

exemplo de um framework para programação do protocolo HTTP com uso do node.js?

Ele facilita a criação de servidores HTTP, rotas, APIs e tudo relacionado à comunicação web. É o framework mais usado no ecossistema Node.

exemplo 1


const express = require('express')
                const app = express()
                
                const cors = require('cors')
                
                // PORT do TCP
                const PORT = 3000 
                
                // endereço IP = 127.0.0.1 do servidor de teste
                const hostname = 'localhost' 
                
                // ------------ Middleware ---------------------
                app.use(express.urlencoded({extended: true}))
                app.use(express.json())
                app.use(cors())
                // ---------------------------------------------
                
                app.get('/', (req,res)=>{
                    res.status(200).json({message: 'Aplicação Rodando!'})
                })
                
                app.get('/login', (req,res)=>{
                    const email_bd = 'carlos@gmail.com'
                    const senha_bd = 123
                
                    const dados = {
                        email: email_bd,
                        senha: senha_bd
                    }
                
                    res.status(200).json(dados)
                })
                
                app.post('/login', (req,res)=>{
                    const email_bd = 'carlos@gmail.com'
                    const senha_bd = 123
                
                    const valores = req.body
                
                    console.log(valores)
                
                    if(email_bd === valores.email){
                        if(senha_bd === valores.senha){
                            return res.status(200).json({ 
                                message: 'login realizado com sucesso!', 
                                nome: valores.nome, 
                                statusLog: 'true'
                            })
                        }else{
                            return res.status(403).json({ message: 'Acesso não autorizado!'})
                        }
                    }else{
                        return res.status(404).json({message: 'Usuário não encontrado'})
                    }
                    
                })
                
                // ---------------- Server ---------------------
                app.listen(PORT, hostname, ()=>{
                    console.log(`http://${hostname}:${PORT}`)
                })
